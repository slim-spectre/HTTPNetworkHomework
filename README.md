# Завдання 1: Дослідження HTTP «вручну» через curl

## Відповіді на питання:

### 1. Яку версію HTTP повернув сервер у відповіді?
Сервер повернув версію HTTP/1.1

### 2. Що означають заголовки у відповіді та запиті?
* Content-Type — показує, у якому форматі прийшли дані. У нашому випадку це application/json (текст у форматі JSON).
* Content-Length — точний розмір отриманих даних у байтах (у нас — 251 байт).
* Server — назва програми (веб-сервера), яка працює на віддаленому комп'ютері та обробила наш запит (gunicorn).
* Date — дата і точний час, коли сервер згенерував відповідь.
* Host — адреса сайту, до якого ми звертаємося. Це потрібно серверу, щоб розуміти, який саме сайт відкрити, якщо на одному IP-адресі їх працює декілька.
* User-Agent — назва програми, через яку робиться запит. Сервер бачить, що ми сидимо не через звичайний браузер, а через консольну утиліту curl.
* Accept — клієнт повідомляє серверу, що він готовий прийняти та обробити будь-який формат відповіді.
* Access-Control-Allow-Origin та Access-Control-Allow-Credentials — налаштування безпеки (CORS). Вони вказують, чи дозволено іншим стороннім сайтам брати дані з цього сервера через JavaScript.

### 3. Що таке Keep-Alive? Навіщо він потрібен порівняно з HTTP/1.0?
* В HTTP/1.0 (стара версія): Для завантаження кожного окремого файлу (картинки, стилю чи тексту) комп'ютер мав створювати нове підключення: підключився  скачав файл  відключився. Через це сайти з великою кількістю елементів сильно гальмували.
* В HTTP/1.1 (режим Keep-Alive): Комп'ютер підключається до сервера один раз і через це єдине відкрите з'єднання завантажує відразу всі потрібні файли по черзі. Це значно прискорює завантаження сторінок та зменшує навантаження на мережу.



## Завдання 2: HTTP методи та коди відповідей

### 1. Таблиця HTTP методів (Заповнена)

| ------- | ----------------------------------------- | -------------- |

| POST    | Створити ресурс / надіслати дані          | Ні             |
| PUT     | Замінити ресурс повністю                  | Так            |

### 2. Таблиця кодів відповіді (Заповнена)

| Код | Назва                 | Коли використовується                            |
| --- | --------------------- | ------------------------------------------------ |
| 200 | OK                    | Успішний запит                                   |
| 201 | Created               | Ресурс створено (POST/PUT)                       |
| 301 | Moved Permanently     | Постійне перенаправлення (назавжди)             |
| 302 | Found (Redirect)      | Тимчасове перенаправлення                        |
| 400 | Bad Request           | Невірний запит від клієнта (помилка в синтаксисі)|

| 500 | Internal Server Error | Помилка на самому сервері (упав код сайту)       |

### Відповіді на питання:

#### 1. Чим відрізняється 301 від 302?
* 301 (Moved Permanently) — це постійне перенаправлення. Сайт переїхав на нову адресу НАЗАВЖДИ. Стара адреса більше ніколи не буде працювати.
* 302 (Found / Temporary Redirect) — це тимчасове перенаправлення. Сайт переїхав на іншу сторінку ТИМЧАСОВО (наприклад, йдуть технічні роботи або акція), але скоро він повернеться на стару адресу.

#### 2. Навіщо браузер кешує 301, але не кешує 302?
* Браузер кешує 301, щоб економити час. Оскільки сайт переїхав назавжди, браузер запам'ятовує це і наступного разу відразу веде тебе на нову адресу, навіть не питаючи старий сервер.
* Браузер НЕ кешує 302, тому що це перенаправлення тимчасове. Якщо його закешувати, то коли сайт повернеться на стару адресу, браузер цього не помітить і продовжить тупо перекидати тебе на тимчасову сторінку.

#### 3. Чим відрізняється 401 від 403?
* 401 Unauthorized — сервер каже: "Я не знаю, хто ти такий. Тобі треба спочатку залогінитися (ввести логін і пароль)".
* 403 Forbidden — сервер каже: "Я знаю, хто ти (ти залогінився), але у тебе немає прав доступу до цієї сторінки". Наприклад, звичайний користувач намагається зайти в панель адміна.
